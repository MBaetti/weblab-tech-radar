== Verteilungssicht

Alle Komponenten werden via Docker Compose als Container auf einem einzelnen Host bereitgestellt.

=== Infrastruktur

[plantuml, target=deployment, format=png]
----
@startuml
!include <C4/C4_Deployment>
<style>
    element {MinimumWidth 190}
</style>

Deployment_Node(host, "Host", "Docker Compose") {

  Deployment_Node(clientContainer, "techradar_client", "nginx:alpine") {
    Container(spa, "Angular", "SPA", "Auslieferung über Nginx")
    Container(proxy, "Reverse Proxy", "Nginx", "Leitet API-Calls an Server weiter")
  }

  Deployment_Node(serverContainer, "techradar_server", "node:20-alpine") {
    Container(api, "Express API", "Node.js", "REST Endpunkte, Prisma ORM")
  }

  Deployment_Node(dbContainer, "techradar_db", "postgres:alpine") {
    ContainerDb(db, "PostgreSQL", "", "Persistierung der Technologie Einträge")
  }
}

Rel(spa, proxy, "Anfragen an /api/")
Rel(proxy, api, "server:3000")
Rel(api, db, "db:5432")

@enduml
----

[cols="1,2" options="header"]
|===
| Container | Beschreibung

| techradar_client
| Nginx liefert die gebaute Angular SPA aus und leitet API-Anfragen als Reverse Proxy an den Server weiter.

| techradar_server
| Node.js Container mit der Express REST-API. Beim Start wird das vordefinierte Datenbankschema und Seed-Daten über Prisma angewendet.

| techradar_db
| PostgreSQL Datenbank. Die Daten werden über ein Docker Volume persistiert.
|===

=== Startvorgang

Der Start des Docker-Compose läuft in folgender Reihenfolge ab:

1. **techradar_db** startet zuerst. Ein Healthcheck stellt sicher, dass die Datenbank bereit ist.
2. **techradar_server** startet nach erfolgreichem Healthcheck der Datenbank. Beim Start werden Schema und Seed-Daten angewendet. Zudem wird die Express-API gestartet.
3. **techradar_client** startet zuletzt und ist über den konfigurierten Port von aussen erreichbar.
