== Laufzeitsicht

=== Technologie erfassen

[plantuml, target=runtime-create, format=png]
----
@startuml
actor "CTO/Tech Lead" as user
participant "Client\n(Angular SPA)" as client
participant "Nginx" as nginx
participant "Server\n(Express API)" as server
database "PostgreSQL" as db

user -> client : Formular ausfüllen und\n"Technologie erfassen" klicken
client -> client : Technologiename, Kategorie und\nBeschreibung prüfen (Pflichtfelder)
client -> nginx : POST /api/technologies
nginx -> server : Weiterleitung
server -> server : Eingabevalidierung
server -> db : INSERT TechnologyEntry
db --> server : Neuer Eintrag
server --> nginx : 201 Created
nginx --> client : JSON Response
client --> user : Zurückgesetztes Formular\nohne Fehlermeldungen

@enduml
----

Der Client sendet die Formulardaten als JSON an die REST-API.
Der Controller validiert alle Pflichtfelder und prüft, ob der Technologiename eindeutig ist bzw. ob er bereits existiert.
Bei ungültigen Daten wird ein `400 Bad Request` zurückgegeben, bei Duplikaten ein `409 Conflict`. Schlägt die Übermittlung oder die Validierung fehl, so zeigt der Client eine entsprechende Fehlermeldung beim Formular an.

=== Technologie publizieren

[plantuml, target=runtime-publish, format=png]
----
@startuml
actor "CTO/Tech Lead" as user
participant "Client\n(Angular SPA)" as client
participant "Nginx" as nginx
participant "Server\n(Express API)" as server
database "PostgreSQL" as db

user -> client : Technologie auswählen,\nFormular ausfüllen\nund "Publizieren" klicken
client -> client : Ring und Einordnung\nprüfen (Pflichtfelder)
client -> nginx : PUT /api/technologies/{id}
nginx -> server : Weiterleitung
server -> server : Eingabevalidierung
server -> server : publicationDate setzen\n(falls erstmalige Publikation)
server -> db : UPDATE TechnologyEntry
db --> server : Aktualisierter Eintrag
server --> nginx : 200 OK
nginx --> client : JSON Response
client --> user : Publikationsdatum anzeigen

@enduml
----

Beim Publizieren sendet der Client einen PUT-Request mit `published: true` sowie den Pflichtfeldern Ring und Einordnung.
Der Controller prüft die Eingaben und setzt bei erstmaliger Publikation automatisch das `publicationDate`.

=== Technologien im Radar anzeigen

[plantuml, target=runtime-view-radar, format=png]
----
@startuml
actor "Mitarbeiter" as user
participant "Client\n(Angular SPA)" as client
participant "Nginx" as nginx
participant "Server\n(Express API)" as server
database "PostgreSQL" as db

user -> client : Radar Viewer öffnen
client -> nginx : GET /api/technologies
nginx -> server : Weiterleitung
server -> db : SELECT * FROM TechnologyEntry
db --> server : Liste aller Einträge
server --> nginx : 200 OK
nginx --> client : JSON Array
client -> client : Filtern nach\npublizierten Einträgen
client -> client : Radar mit\nTechnologiedaten rendern
client --> user : Radar Visualisierung

@enduml
----

Beim Öffnen des Viewers lädt der Client alle Technologien über die REST-API.
Anschliessend werden nur publizierte Einträge gefiltert und als Radar dargestellt.

=== Technologie löschen

[plantuml, target=runtime-delete, format=png]
----
@startuml
actor "CTO/Tech Lead" as user
participant "Client\n(Angular SPA)" as client
participant "Nginx" as nginx
participant "Server\n(Express API)" as server
database "PostgreSQL" as db

user -> client : Technologie auswählen\nund "Löschen" klicken
client -> nginx : DELETE /api/technologies/{id}
nginx -> server : Weiterleitung
server -> db : SELECT TechnologyEntry
db --> server : Eintrag gefunden
server -> db : DELETE TechnologyEntry
db --> server : Gelöschter Eintrag
server --> nginx : 200 OK
nginx --> client : JSON Response
client --> user : Technologie aus\nListe entfernen

@enduml
----

Beim Löschen sendet der Client einen DELETE-Request mit der ID der Technologie.
Nach erfolgreichem Löschen wird die Technologie aus der Liste im Client entfernt.
