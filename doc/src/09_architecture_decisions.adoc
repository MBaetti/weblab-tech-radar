[[section-design-decisions]]
== Architekturentscheidungen

[cols="1,2,2" options="header"]
|===
| Entscheidung | Begründung | Verworfene Alternativen

| Monorepo mit Client und Server
| Einfaches Setup, Entwicklung und Deployment
| Multirepo: höherer Konfigurationsaufwand ohne Mehrwert

| Prisma ORM
| Typensicher, deklaratives Schema, integriertes Seeding und intuitive Abfragen
| Plain Querries: mehr Boilerplate, keine Typensicherheit, weniger intuitiv

| REST API
| Einfache CRUD-Operationen über Endpunkte, gut unterstützt von Angular HttpClient
| REST API mit OpenAPI und Swagger: zu viel Aufwand für dieses Projekt
|===
