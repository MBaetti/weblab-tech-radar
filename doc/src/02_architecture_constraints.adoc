[[section-architecture-constraints]]
== Randbedingungen

Der Weblab Tech-Radar soll:

* die im vorherigen Kapitel beschriebenen Must-Anforderungen erfüllen.
* als Angular SPA mit Angular Material im Frontend realisiert werden.
* mit Next.js, Express für die API und Prisma als ORM im Backend umgesetzt werden.
* TypeScript als Programmiersprache verwenden.
* eine PostgreSQL Datenbank zur Datenhaltung verwenden.
* vollständig über Docker Compose containerisiert und auslieferbar sein.
* über .env-Dateien konfigurierbar sein.
* mit Vitest für Unit-Tests und Cypress für E2E-Tests automatisiert getestet werden.
* nach arc42 dokumentiert werden.
* rechtzeitig zum Abgabetermin am 6. März 2026 mit den geforderten Dokumenten abgegeben werden.
